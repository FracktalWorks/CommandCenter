## 1. Why this doc / where to continue

The observability **Office** view renders each agent as a layered pixel-art scene
in a themed room (working / sleeping / error), plus a **war room** for multi-agent
collaboration. Today those sprites are **procedural placeholders**. The plan is to
replace the placeholder layers with real pixel art generated by **Pixel Lab**
(pixel-art-native, transparent output, animation frames — the right tool for
character/prop sprites; Higgsfield is for room/background *scenes*).

**Blocker on THIS environment:** outbound to `pixellab.ai` returns proxy **403**
(org egress policy). The proxy README says report it, don't route around it. So:
continue on **SPC** (or any host that can reach `pixellab.ai`), where the API key
works. The engine, the swap seam, and this spec are all on `main`, so the next
session only has to *generate assets to spec and wire them*.

## 3. The composition model (how a look is assembled)

```ts
interface AvatarConfig {
  skin: string;
  hair: { style: "spiky"|"bun"|"short"|"long"; color: string };
  outfit: { type: "hoodie"|"suit"|"sweater"; color: string; color2: string };
  accessory: "glasses"|"headset"|"beanie"|null;
  deskProps: string[];              // "mug"|"plant"|"papers"|"phone"|"dual-monitor"
  room: RoomTheme; wallProp: "window"|"board"|"whiteboard";
  screen, screen2, accentA, accentA2, desk, desk2: string;
}
```

- **`deriveAvatar(name, override?)`** — `roleFor(name)` keyword-matches a role
  (coder/sales/planner/triage/reconciler/orchestrator) → a **preset signature**
  (room + outfit + accessory + props), then adds **per-agent variation** (skin,
  hair style/colour) from a name hash. Zero-config fitting avatars; `override`
  pins any field (this is where **backend avatar config** will feed in).
- **`AgentScene`** composes `room() + character() + desk()` as z-ordered
  **descriptor rects**; `state` ∈ `working|idle|error`. Animation via `.sc-*`
  CSS classes (`SCENE_STYLE`): typing hands, head bob, blink, screen flicker,
  mug steam; sleeping = slump + breathe + floating Zzz; error = shake.
- **`WarRoomScene({ names })`** — conference room + seated agents (their derived
  palettes) + presentation screen + chatter.

## 4. THE ASSET SPEC — the mix-and-match contract (generate to THIS)

For externally-generated sprites to compose as layers, they must obey:

- **Cell:** one canonical canvas — **64 × 56** logical px for character scenes
  (matches `AgentScene`'s `viewBox="0 0 64 56"`); integer-scaled; **transparent
  PNG (RGBA)**.
- **View:** front-facing, **seated, waist-up**, centered on a shared **anchor
  grid**.
- **Anchors** (coords in the 64×56 grid, read from `scene.tsx`):
  - head box ≈ `(27,11)`–`(37,21)`, head center ≈ `(32,15)`; eyes ≈ x29 & x33, y15
  - neck ≈ `(29,20)`; shoulders/torso ≈ `(24,22)`–`(40,34)`
  - hands on keyboard: **L** ≈ `(25,36)` 4×2, **R** ≈ `(35,36)` 4×2
  - monitor ≈ `(20,20)`–`(36,32)`; keyboard ≈ `(24,37)` 16×2
  - desk (fg) ≈ `(6,38)` 52×6; rug ≈ `(10,40)`; wall prop ≈ `(40,5)` 20×15
- **Layers (z-order):** `room bg → rug → chair → torso/outfit → arms/hands(anim)
  → head/skin → face(anim) → hair → accessory → desk props → desk(fg)`.
- **One transparent PNG per layer** on the full cell (so they simply stack), OR a
  tight sprite + a declared anchor offset in the manifest.
- **Recolor:** author hair/outfit/skin either as **colour variants** or as a
  **flat tintable base** (generate on neutral grey; tint in-app). Prefer tintable
  to keep the set small.
- **Animation:** animated layers (typing hands, blink) as a **horizontal frame
  strip** (2–4 frames); the manifest declares `frames` + `fps` + `state`.
- **Environments:** room **tiles on a 16px grid**; a room = a tile-id 2D array +
  furniture props (depth-sorted).
- **Manifest (`manifest.json`):** registers every part
  `{ layer, file, anchor:{x,y,w,h}, frames, fps, recolor, roleTags }`, plus rooms
  and palettes. **This is the single thing the app loads** — procedural and real
  art become interchangeable.

## 5. The swap seam — how to plug a real sprite in

Each layer in `scene.tsx` is a list of **descriptor rects** rendered by
`renderDesc()`. Two ways to swap in real art:

1. **Per-layer `<image>`:** replace a layer's rects with a single SVG
   `<image href={dataUri} x=… y=… width=… height=…/>` anchored to the same
   coords. Keep the layer's group class (`.sc-head`, `.sc-hand-l`, …) so the
   existing animation still applies.
2. **Manifest-driven (recommended):** add a `partSprite(layer, config, state)`
   branch point — if the manifest has a `file` for that (layer, variant), emit
   `<image>` (or a frame-strip `<image>` animated via CSS `steps()`); else fall
   back to the procedural rects. One switch flips the whole office from
   placeholder → real art with no page changes.

`AgentScene` already accepts an optional `config` and there is a documented
`<PixelWorker src=…>`-style seam note at the top of `scene.tsx`.

## 6. Pixel Lab — generation plan (run on SPC)

- **Auth/key:** store the Pixel Lab API key in an env var (e.g.
  `PIXELLAB_API_KEY`) — **never commit it**. Check Pixel Lab's current docs for
  the generate endpoint + auth header.
- **Generate to the ASSET SPEC:** transparent background, small dimensions,
  isolated element per layer (e.g. "hoodie torso, front view, transparent
  background"), or full-character variants if per-part isn't practical (Pixel Lab
  supports character + rotation + animation frames).
- **Shopping list (layers):** bodies (tintable) · hair (styles × tintable) ·
  outfits (hoodie / suit / sweater / lab-coat / vest) · accessories (glasses /
  headset / beanie / halo) · **hand-typing 2–4 frame strip** · desk props (mug /
  plant / papers / phone / books / dual-monitor) · **room tilesets** (dev-den,
  sales-floor, planning-room: wall / floor / rug / window / board / table).
- **Embed:** base64 the PNGs → data-URIs in a generated `assets` module (CSP-safe
  everywhere), or drop static files under `public/` and reference by URL. Wire via
  the manifest/seam.
- **Validate before committing:** render each generated layer **headless with the
  bundled Playwright/Chromium** (the workflow used throughout this feature —
  `/opt/pw-browsers/chromium-*/chrome-linux/chrome`, `NODE_PATH` = control_plane
  node_modules) and eyeball it. Don't ship art blind.

## 7. Higgsfield

`.mcp.json` carries a `higgsfield` HTTP MCP entry, but it **needs OAuth/auth**
(not connected here). Use Higgsfield for **room/background scene images** (single
images, no transparency/layering needed — where diffusion shines); use **Pixel
Lab** for the mix-and-match **character/prop sprite layers**.
