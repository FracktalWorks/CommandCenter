The Architecture Overview
The system follows a simple but powerful flow:


User → Copilot SDK → Router Prompt → Selected Foundry Agent → Output → Finalizer → User
Each component has a clear responsibility:

Copilot SDK orchestrates the entire workflow
Router Prompt decides which agent should handle the request
Foundry Agents perform the actual work
Finalizer produces a polished, user‑friendly answer
This architecture is intentionally minimal. The simplicity is what makes it scalable.

The Router Prompt: The Brain of the System
The router prompt is where the magic happens. It forces the model to choose exactly one tool.

Here's a simplified version:


def build_router_prompt(user_text, tools):
    return f"""
You are a routing controller...."""
This prompt ensures:

the model cannot call multiple tools
the model cannot hallucinate extra keys
the model must justify its decision
the output is always structured JSON
This is the core of multi‑agent orchestration.

The Router Decision Logic
The router decision is handled through the Copilot SDK session:


resp = await session.send_and_wait({"prompt": prompt})
decision = json.loads(resp.data.content)
If the router returns invalid JSON, the system defaults to:


{"tool_name": "none", "args": {}, "reason": "invalid JSON"}
This ensures the system never crashes due to malformed output.

The Finalizer: Polishing the Output
After the agent returns its output, the finalizer produces a clean response:


final_prompt = f"""

The finalizer ensures:

consistent tone
error handling
improved clarity
merged reasoning + tool output
This is the Copilot experience layer.

Lessons Learned While Building This
Router prompts are extremely powerful: A well‑designed router prompt can outperform complex routing logic.
Foundry Projects make modularity natural: Each project becomes a self‑contained capability.
Copilot SDK simplifies orchestration: It handles sessions, tool calling, and finalization.
Multi‑agent systems feel intuitive: Once you build one, you realize this is how AI should work.
Separation of concerns leads to cleaner architecture: Each agent does one thing well. The orchestrator coordinates. The finalizer polishes.
This pattern is scalable, maintainable, and future‑proof.
