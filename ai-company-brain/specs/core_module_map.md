## State-of-the-art comparison vs [awesome-harness-engineering](https://github.com/ai-boost/awesome-harness-engineering) (2026-07-02)

Reviewed the A1+A2+D1 work (and adjacent modules) against the list's named
practices. Verdict per area — **=SOTA** (matches/exceeds the documented best
practice), **~solid** (correct but with a known gap), **gap** (below best
practice):

| List practice (exact wording) | Our implementation | Verdict |
|---|---|---|
| **"hibernate-and-wake" persistence** · "Checkpointing: Resumption across context windows" (Agent Loop) | Redis-Stream event log per thread is the source of truth; `run_detached` decouples execution from the HTTP response; reconnect replays from the log + follows live. A run survives browser close AND (P1-2) cross-worker control. | **=SOTA** — this is the "decouple the brain from the hands" pattern implemented end-to-end, eval-locked. |
| **"Real-time streaming: AG-UI protocol for agent-to-UI communication with state updates"** (HITL) · **"Event-driven: Multi-agent streaming with shared event bus"** (Orchestration) | One canonical AG-UI translator (`event_translator.py`) feeds all four executor paths; Phase 3b makes it message-id-native (real segment boundaries, not inferred). Redis Stream = the shared event bus; SUB_AGENT_* forwarded. | **=SOTA** for the protocol + single-translator discipline; the list treats AG-UI as the reference and we implement it with parity evals. |
| **"HITL interrupts: Pause agent execution for human review"** · **"Approval flows: Structured permissions before destructive actions"** (HITL) | Blocking-Future HITL (ask_user/ask_questions/request_confirmation); confirmations **fail closed** for destructive/irreversible actions (HH-2); P1-2 makes the answer reach the owning worker. | **=SOTA** on fail-closed + real interrupt-and-resume (many harnesses only queue a reply); ~gap: 3 delivery paths is internal complexity debt. |
| **"Beyond Permission Prompts—structured authorization"** · tool annotations `readOnlyHint`/`destructiveHint`/`idempotentHint`/`openWorldHint` (Permissions) | `tool_annotations.py` risk vocabulary + `own_tool_scope`; confirmations gate on `is_destructive`. | **~solid** — vocabulary + fail-closed shipped; **gap**: no allowlist state-machine per workflow phase, and normal runs are in-process (B6=C, the real weak spot). |
| **"Track all agent decisions for regression detection"** · **"Eval harnesses: measure agent behavior, not just outputs"** (Verification) | Trajectory evals are CI-blocking and cover the harness invariants (translator parity, fold, persistence, cross-worker control, watchdog, loop detection). | **~solid** — the *harness* is eval-locked (rare + strong); **gap**: no agent-*quality* evals yet (behavior/outcome scoring), which is the list's emphasis. |
| **Loop / failure detection** (implied by "verify" stage + regression tracking) | `LoopDetector` trips a RUN_ERROR on identical repeated tool calls; unified `WatchdogPolicy` bounds idle/tool/HITL stalls. | **=SOTA** — explicit loop + stall guards with one inspectable policy; most harnesses rely on a bare wall-clock timeout. |
| **"Cross-session: Persistent planning documents"** · **"LLM-curated memory with learned relevance filtering"** (Memory) | Mem0 (pgvector) episodic + Graphiti bi-temporal KG + NOTES.md; P1-9 extracts at the run boundary (survives reconnect, includes the answer). | **~solid** — dual-store memory + run-boundary extraction is ahead of typical; **gap**: no write-gate on Graphiti (writes every turn), no learned relevance filter. |
| **"Context as constraint" · "Compaction (84% token reduction)"** (Context) | Auto-compaction at 80% of the real window w/ hysteresis + checkpoint model. | **~solid**; **gap**: frontend-only (API/webhook runs never compact); history assembly still client-side (C2=C+). |
| **Prompt Caching** (byte-stable prefixes) | Only `lru_cache` on the registry block; the caching plan is written but unbuilt. | **gap** — C4=D; the single biggest cost lever still open (now measurable via HH-3 counters). |
| **Multi-worker / distributed correctness** (Governing agents; implied by managed-agent scaling) | P1-2 cross-worker control bus (Redis pub/sub) + Postgres-durable session store → cancel/HITL/session survive across workers. | **=SOTA** — the list gestures at "decouple brain from hands"; we implement the distributed-systems corollary (control commands route to the owning worker) explicitly. |

**Bottom line:** on the **agent loop, streaming/event protocol, HITL, long-running + multi-worker correctness, and loop/stall detection** — the A1/A2/A3/A5 core — CommandCenter is **at or above the documented state of the art**, and unusually, the harness itself is locked by CI-blocking trajectory evals (a practice the list preaches but few implement). The **genuine gaps are all outside the A1+A2+D1 core**: prompt caching (C4=D, top cost lever), sandboxing/isolation (B6=C, top safety lever), central context assembly (C2=C+), and agent-*quality* evals (E1). None are correctness bugs; all are known and scheduled in the review order.
